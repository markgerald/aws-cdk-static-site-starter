# Repository Guidelines

## Project Structure & Module Organization

This repository is an AWS CDK v2 TypeScript project for a static website on private S3, CloudFront OAC, ACM, and Cloudflare DNS.

- `bin/app.ts` is the CDK entry point; keep it limited to app composition and tagging.
- `lib/config/` resolves context and environment defaults.
- `lib/stacks/` contains stack classes; `lib/constructs/` contains reusable resource factories; `lib/outputs/` contains outputs.
- `lib/static-site-stack.ts` is a compatibility re-export for existing imports.
- `test/*.test.ts` contains Jest assertions for config, workflows, and CloudFormation templates.
- `website_src/` contains React/Vite source; `website_dist/` is generated by `npm run build:site`.
- `.github/workflows/ci.yml` runs CDK build, site build, tests, and `cdk synth`; `deploy.yml` syncs `website_dist/`.

## Build, Test, and Development Commands

Use Node.js 22 or newer.

- `npm ci` installs locked dependencies for clean local or CI runs.
- `npm run build` compiles TypeScript with `tsc` into `dist/`.
- `npm run build:site` builds the React site into `website_dist/`.
- `npm run build:all` compiles CDK and builds the site.
- `npm test` runs the Jest CDK unit tests.
- `npm run test:coverage` runs Jest with coverage reporting.
- `npm run watch` starts TypeScript in watch mode.
- `npm run synth` runs `cdk synth` to generate CloudFormation templates.
- `npm run deploy` deploys all stacks; pass context values when needed, for example `npx cdk deploy --all --context domainName=example.com`.

## Coding Style & Naming Conventions

TypeScript is strict (`strict`, `noImplicitAny`, `noImplicitReturns`, `alwaysStrict`). Use two-space indentation, single quotes, semicolons, and explicit interfaces for stack and factory props. Keep CDK construct IDs stable, such as `WebsiteBucket`, `Distribution`, and `OriginAccessControl`. Prefer context-driven configuration in `lib/config/`.

## Testing Guidelines

Tests use Jest with `ts-jest` and match `**/test/**/*.test.ts`. Update assertions when resources, IAM policies, CloudFront behavior, or context defaults change. Before a PR, run `npm run build`, `npm test`, and `npm run synth`.

## Commit & Pull Request Guidelines

Git history is not present in this checkout, so use concise imperative commit subjects, for example `Add SPA fallback tests`. Pull requests should describe infrastructure changes, list validation commands, link issues, and include screenshots or URLs when `website_src/` changes. Call out AWS, Cloudflare, ACM, or Actions variable changes.

## Security & Configuration Tips

Do not commit AWS credentials, Cloudflare tokens, generated `cdk.out/`, or local `.env` files. Keep ACM certificates for CloudFront in `us-east-1`. Start Cloudflare records as DNS only, and avoid adding Route 53 hosted-zone configuration unless the architecture changes intentionally.
